Когда вы используете команду "go mod init <module-name>", вы фактически создаете новый модуль и создаете файл "go.mod" для этого модуля.
Команда "go mod init" задает имя модуля (в "go.mod") и инициализирует модульную систему.

Если вы создаете новый проект и хотите использовать модули, вам нужно сначала инициализировать модуль командой "go mod init", что создаст файл "go.mod".
Без файла go.mod проект не будет считаться модулем Go, и система модулей не будет управлять зависимостями.

Файл go.mod — это основной способ определения и управления модулем в Go. Он содержит ключевую информацию для модульной системы Go.

# Команда "go mod init <module_name>" создает файл "go.mod" (инициализирует новый модуль), если его еще нет.
# ДЛЯ СРАВНЕНИЯ: "go mod download" загружает зависимости, указанные в go.mod

# Команда "go mod tidy" используется для автоматического управления зависимостями на основе анализа исходного кода.
# "go mod tidy" анализирует код и обновляет go.mod, а также создает или обновляет go.sum.

# "go.sum" - это файл, который содержит контрольные суммы для всех модулей (и их версий), которые указаны в go.mod и были загружены в ваш проект.
#  Он обеспечивает безопасность и целостность зависимостей, помогая гарантировать, что версии зависимостей не изменяются неожиданным образом.
